Troubleshooting

Connection Never Establishes

ProblemSolution
Hostname doesn't resolveCheck DNS or use IP address directly
Port 22 is closedEnsure SSH server is running: sudo systemctl status ssh
Firewall blockingCheck firewall rules: sudo ufw status
Key not foundVerify the key path exists: ls -la ~/.ssh/
Key not authorizedAdd your public key to remote ~/.ssh/authorized_keys
Wrong SSH portVerify the port in Settings matches the remote SSH server config

Connection Drops Frequently

  • Check network stability (Wi-Fi interference, Ethernet cable)
  • AutoSSH will automatically reconnect with backoff — wait up to 60 seconds
  • Check SSH server ClientAliveInterval and ClientAliveCountMax settings
  • Check if the remote device is going to sleep or losing power

Logs

Run AutoSSH from a terminal to see logs:

RUST_LOG=info ./auto-ssh

Available log levels: error, warn, info, debug, trace

App Doesn't Start

  • Ensure all system dependencies are installed (libssl3, libc6)
  • Run from terminal and check the error message
  • Try removing the config file: rm ~/.config/auto-ssh/config.toml
  • Check if another instance is already running
