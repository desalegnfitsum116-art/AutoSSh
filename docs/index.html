Introduction

AutoSSH is a lightweight desktop utility that automatically connects trusted devices through SSH when both devices are powered on and reachable on the network. No terminals, no manual commands.

Rust MIT License Cross Platform v0.1.0

Features

  • Auto-connect — Establishes an SSH session as soon as the remote device becomes reachable
  • Auto-reconnect — Survives network drops, sleep/wake cycles, and remote reboots with exponential backoff
  • Zero terminal usage — No need to open a shell and type SSH commands
  • SSH key authentication — Uses your existing Ed25519 or RSA keys
  • Background service — Runs quietly in the system tray and monitors devices 24/7
  • Live status — Dashboard shows Offline → Online → SSH Ready → Connected
  • System tray integration — Connect, disconnect, or quit directly from the tray menu
  • Native GUI — Built with egui/eframe — lightweight, dark-themed
  • Start on boot — Optionally launch at login and minimize to tray
  • Portable config — Settings stored in TOML format at ~/.config/auto-ssh/config.toml

Use Cases

  • Developers who want to SSH into a home server without typing ssh user@host every time
  • Homelab users who need persistent terminal access to a Raspberry Pi
  • DevOps engineers managing jump-box access
  • Anyone managing multiple computers who wants SSH connections that just work

Quick Start

  1. Launch AutoSSH
  2. Open Settings (gear icon in the bottom-right)
  3. Enter the remote host IP or hostname
  4. Enter the SSH username
  5. Set the SSH port (default: 22)
  6. Provide the path to your SSH private key (e.g. ~/.ssh/id_ed25519)
  7. Toggle Auto-Connect to Enabled
  8. Click Save

AutoSSH immediately starts monitoring the remote device. When the device becomes reachable, connection happens automatically.

Offline Host unreachable or DNS fails to resolve
→
Online Host resolves but SSH port is closed
→
SSH Ready Port 22 is open and accepting connections
→
Connected SSH session established and authenticated

Configuration

Configuration is stored at ~/.config/auto-ssh/config.toml. You can edit it directly or use the Settings panel in the GUI.

Example Config

device_name = "Laptop"
remote_host = "192.168.1.10"
username = "fitsum"
port = 22
ssh_key_path = "~/.ssh/id_ed25519"
auto_connect = true
start_on_boot = true
poll_interval_seconds = 5

Usage Guide

Dashboard

The main dashboard shows:

  • Local Device — Always shows Online (your machine)
  • Remote Device — Current status of the remote device
  • Auto-Connect — Whether auto-connect is enabled (green/gray)
  • Last Connection — Timestamp of the last successful SSH connection
  • Connect / Disconnect — Manual connection controls

System Tray

  • Connect/Disconnect — Toggle SSH connection
  • Show Dashboard — Bring the main window to front
  • Quit — Exit the application entirely

The tray icon changes color based on connection status. Clicking the window close button hides it to the tray instead of quitting.

Settings

Access settings from the gear icon on the dashboard. Changes take effect after clicking Save. The monitor thread automatically picks up new host/port settings.

Auto-Connect Rules

IF device is online
  AND SSH port is reachable
  AND SSH key authenticates successfully
THEN connect automatically.

IF connection drops
THEN retry with exponential backoff
   (1s → 2s → 4s → 8s → 16s → 32s → 60s caps)

Security

SSH Key Authentication

AutoSSH uses SSH key authentication exclusively. No passwords are stored, transmitted, or handled by the application. This is the most secure authentication method for automated SSH connections.

Credential Storage

  • Configuration is stored in your home directory (~/.config/auto-ssh/config.toml)
  • File permissions: 644 (world-readable) — ensure your SSH private key has chmod 600
  • No credentials are sent over the network beyond standard SSH protocol

Network Security

  • Outbound connections only to the configured remote host and port
  • All traffic is encrypted via the SSH protocol (libssh2)
  • No telemetry, analytics, or external service calls
  • No listening ports or server functionality

Key Permissions

chmod 600 ~/.ssh/id_ed25519
chmod 700 ~/.ssh
Important: Ensure your SSH private key file has restricted permissions. AutoSSH will refuse to use a key with overly permissive permissions (libssh2 enforces this).

Troubleshooting

Connection Never Establishes

ProblemSolution
Hostname doesn't resolveCheck DNS or use IP address directly
Port 22 is closedEnsure SSH server is running: sudo systemctl status ssh
Firewall blockingCheck firewall rules: sudo ufw status
Key not foundVerify the key path exists: ls -la ~/.ssh/
Key not authorizedAdd your public key to remote ~/.ssh/authorized_keys

Connection Drops Frequently

  • Check network stability (Wi-Fi interference, Ethernet cable)
  • AutoSSH will automatically reconnect with backoff — wait up to 60 seconds
  • Check SSH server ClientAliveInterval and ClientAliveCountMax settings

Logs

Run AutoSSH from a terminal to see logs:

RUST_LOG=info ./auto-ssh

Log levels: error, warn, info, debug, trace

FAQ

Do I need root / sudo permissions?

No. AutoSSH runs as a regular user and uses standard SSH keys.

Can it handle password-based auth?

Not currently. SSH key authentication is preferred and more secure.

What happens when my laptop goes to sleep?

AutoSSH detects the connection drop, enters reconnection mode, and automatically reconnects when the network is available again using exponential backoff.

Does it work with non-standard SSH ports?

Yes. Set the port in Settings or directly in config.toml.

Is this a terminal emulator?

No. AutoSSH establishes SSH sessions in the background. It does not provide a terminal UI or shell.

Can I use it with SSH jump hosts?

Not yet. This is planned for a future release.

How do I uninstall?

If installed via .deb: sudo apt remove auto-ssh. Remove the config: rm -rf ~/.config/auto-ssh. Remove autostart: rm ~/.config/autostart/auto-ssh.desktop.
